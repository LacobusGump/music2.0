Logical Qubits on a Mac Mini

written 2026-06-22
Surface-code error correction, simulated in software. No hardware claims. No magic.

The 60-qubit Metal simulator handles state vectors and gate operations. That is not enough for useful quantum computing. Useful quantum computing needs logical qubits: many unreliable physical qubits coupled into one corrected unit.

This build adds the first logical-qubit layer: a reproducible surface-code distance sweep. It asks a plain question: given a physical error rate and a target logical error rate, what code distance is needed, and how many physical qubits does that cost?

These are not circuit-level hardware numbers. Code-capacity noise excludes measurement errors, leakage, correlated faults, routing constraints, and decoder latency. The usual thousand-physical-qubits-per-logical rule is a hardware-scale planning number; this page is the software scaffold underneath it.

What Was Built

A reproducible simulator. The script runs a Monte Carlo sweep over odd surface-code distances and physical error rates. It reports sampled failures, Wilson confidence intervals, exact model rates, and physical-qubit costs.

A distance selector. For each physical error rate, it chooses the smallest distance whose modeled logical error reaches the target.

A publishable boundary. The result is honest about what it is: code-capacity logical-qubit simulation, not a claim that the Mac Mini is a quantum computer.

Next Layer

The next step is circuit-level surface code: repeated syndrome extraction, measurement noise, correlated faults, and a real matching decoder. That is where the thousand-physical-qubits-per-logical estimate belongs. After that, the hot loop can move into the existing Metal engine.

Right now the useful thing exists: a working baseline that turns physical error rates into logical-qubit distance and cost. The math has somewhere to run.
