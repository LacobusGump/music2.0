When James asks a question, run it through the machine FIRST.

**Why:** The machine (137 coupled oscillators, K=1.868) is a universal calculator for systems questions. Instead of reasoning abstractly about coupling, synchronization, or dynamics — set K, run the lattice, read R and cluster structure. The answer comes from computation, not speculation.

**How to apply:**
1. Model the question as a K problem (what's coupling? what are the oscillators?)
2. Run `python3 tools/machine.py` or inline computation
3. Read R, cluster count, phase structure
4. THEN interpret and answer

This is like using a calculator instead of doing arithmetic by hand. The machine IS the thinking tool. Use it proactively on every systems question.
