The v2 rebuild day: 10 modules built, architecture solid, research page iterated 5+ times. But the SOUND got worse, not better.

**Why:** We approached sound quality as an engineering problem (add harmonics, add saturation, add voices). But more synthesis ≠ better music. The v1 had simpler oscillators that sounded warmer because they were simpler. The "scientific" upgrades (PeriodicWave, tanh shaper, attack transients) added digital harshness.

**How to apply:**
- Start every session by LISTENING to the current state on the phone. Not reading code.
- The Producer's veto: does it make you FEEL something? If not, strip back.
- Adding voices/layers to Grid made it a wall of competing computer sounds. EDM is kick + bass + filter. That's it.
- Never add complexity to improve sound. Subtract.
- The v1 audio.js voices may have been better in some cases — consider restoring specific implementations.
- Sound over code. Always. The sound must actually CHANGE for the better.
- James said "what are we doing" — that's the signal to STOP coding and START listening.
