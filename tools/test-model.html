Reversibility Test Model

Everything reversible except prime erasures

Four computational tests. If the simulation matches the theory at every point, the framework is validated. If there is a discrepancy, we find the bug — in the theory or the code. Test 4 is the make-or-break: does structure dominate size?

Test 1: Sorting — Standard vs Reversible

Standard comparison sort erases 1 bit per swap (you lose "which was bigger"). Bennett-style reversible sort keeps all intermediates, then erases only the minimum at the end. The gap is pure thermodynamic waste.

The framework says: most of what standard computing erases is not structurally necessary. Only the prime-decomposition information — the irreducible structure — demands irreversible erasure.

Test 4: Adder vs Multiplier — Degeneracy Determines Cost

The make-or-break test. Same output number n, two different operations. Addition has degeneracy ~n. Multiplication has degeneracy d(n). If structure matters, these must give different Landauer bounds — and they do.

The Ratio: How Much Less Does Multiplication “Cost”?

Ratio = ln(d(n)) / ln(n−1). Lower = more structured, less degeneracy. Primes hit the floor. Highly composite numbers peak.

The old claim: “primes cost something.” The real claim: lost degeneracy costs something. Primes are zero-degeneracy endpoints — the only numbers where multiplication has minimal ambiguity. The energy tax is kT times “how many ways could reality have produced what you kept.”
