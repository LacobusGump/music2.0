Entropy-Aware Architecture Simulator

Does factorizing erasure lead to better designs?

Computation destroys information at specific moments. This simulator compares schedules for the same computation by logical erasure, memory, operations, and garbage — then shows the Pareto frontier between them. No physical energy claims. Just the structure of irreversibility, made visible.

Search: (N−1)·ln(2) vs ln(N)

Find the index of a target in an unsorted list of N items. Conventional search discards each failed comparison — one bit erased per check. Entropy-aware search does reversible comparisons (each stored as a temporary comparison register), copies the match index, uncomputes all comparison state, then erases only the final position uncertainty: log₂(N) bits.

Conventional wins on simplicity (fewer total ops, zero garbage). Entropy-aware wins on logical erasure (ln(N) vs (N−1)·ln(2)). The gap: information that was created and then discarded, instead of preserved and uncomputed.

Sorting: Conventional vs Entropy-Aware

Comparison sort erases 1 bit per comparison (which was bigger?). Entropy-aware sort uses reversible comparisons, copies the sorted output, uncomputes all intermediates, then erases only the permutation information: ln(N!) nats, factorized through the primes of N!.

Conventional wins on memory and latency. Entropy-aware wins on logical erasure. The tradeoff grows with N — and so does the design space.

Multiply: Where Does the Cost Live?

Compute a × b = n, then discard inputs. Both versions erase the same total: ln(n) nats. The factorized version doesn't erase less — it reveals where the information goes. Which prime absorbed which bits? That's the structure the generic version throws away.

Pareto Frontier: Memory vs Logical Erasure

The best schedule depends on the objective, not ideology. Drag the sliders: entropy-aware wins on erasure, conventional wins on simplicity, hybrids win in between. That's the frontier. Choose what matters.

This simulator treats information destruction as a schedulable resource. The question is not “can we erase less” but “where on the frontier do we want to be.”

Logical erasure depends on the task specification, prior assumptions, and representation. This tool compares schedules within an explicit cost model; it does not claim a universal physical energy law.
