computation ≠ destruction

This compiler rewrites computations into different schedules that trade off information destruction, memory, and operations. It does not eliminate necessary erasure. It eliminates premature erasure — information destroyed before the computation proves it was unneeded. The compiler identifies every destructive operation, lifts reversible structure, inserts uncompute points, plans collapse events, and generates multiple schedules on the Pareto frontier.

Per-op, the rewrite preserves the same logical content (1.0 bits deferred, not eliminated). But when applied across the full computation, global correlation between operations means the total final collapse can be far smaller than the sum of individual ops. 64 comparisons each erase 1 bit independently — but they are all answering the same question (which position?), so the minimum final collapse is log₂(64) = 6 bits. The local rewrite enables this; the global analysis realizes it.
